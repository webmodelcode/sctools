# Quick Git Commit for Staged Files

Follow these steps to create a commit for staged files.

1. **Check GIT Status**
   Check if there are staged files to commit.

   ```bash
   git status --porcelain
   ```

   If there are no staged files (lines starting with 'M', 'A', 'D', etc., in the first column), **STOP** and inform the user that they need to stage files using `git add` first. Do NOT automatically stage files.

2. **Analyze Changes**
   Read the staged changes to understand what is being committed.

   ```bash
   git diff --cached
   ```

3. **Generate Commit Message**
   Create a commit message following the **Conventional Commits** format with emojis.

   **Format**: `<emoji> <type>: <description>`

   **Rules**:
   - **Present tense, imperative mood**: "add feature" NOT "added feature".
   - **Concise**: Keep the first line under 72 characters.
   - **Atomic**: If changes cover multiple distinct logic parts, suggest splitting the commit (but proceed with one if the user insists or if they are reasonably related).
   - **No Signatures**: Do not add "Generated by..." footers.

   **Emoji & Type Reference**:
   - ✨ feat: A new feature
   - 🐛 fix: Bug fix
   - 📝 docs: Documentation changes
   - 💄 style: Code style changes (formatting, etc)
   - ♻️ refactor: Code refactoring
   - ⚡️ perf: Performance improvements
   - ✅ test: Adding or fixing tests
   - 🔧 chore: Changes to the build process, tools, etc.
   - 🚀 ci: CI/CD improvements
   - 🗑️ revert: Reverting changes
   - 🧪 test: Add a failing test
   - 🚨 fix: Fix compiler/linter warnings
   - 🔒️ fix: Fix security issues
   - 👥 chore: Add or update contributors
   - 🚚 refactor: Move or rename resources
   - 🏗️ refactor: Make architectural changes
   - 🔀 chore: Merge branches
   - 📦️ chore: Add or update compiled files or packages
   - ➕ chore: Add a dependency
   - ➖ chore: Remove a dependency
   - 🌱 chore: Add or update seed files
   - 🧑‍💻 chore: Improve developer experience
   - 🩹 fix: Simple fix for a non-critical issue
   - 🎨 style: Improve structure/format of the code
   - 🚧 wip: Work in progress
   - 👷 ci: Add or update CI build system
   - 💥 feat: Introduce breaking changes
   - 🗃️ db: Perform database related changes
   - 🦺 feat: Add or update code related to validation

4. **Execute Commit**
   Propose the commit command with the generated message.

   ```bash
   git commit -m "your message here"
   ```

   _Note: If specific detailed description is needed, use multiple `-m` or newlines._

5. **Verify**
   After committing, you can show the last commit to confirm.
   ```bash
   git log -1 --stat
   ```
